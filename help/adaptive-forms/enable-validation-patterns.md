# Adobe Experience Manager アダプティブフォームでテキスト入力検証パターンを有効にして使用

## 概要

Adobe Experience Manager フォームでは、テキスト入力コンポーネントの特定の検証フォーマット（社会保障番号、電子メールアドレス、郵便番号など）がテンプレートポリシーレベルで制御されます。このガイドでは、次の方法について説明します。

1. テンプレートエディターにアクセスし、テキスト入力コンポーネントのポリシーを設定して、検証パターンを公開します。
2. アダプティブフォームを作成し、それらの検証パターンをテキスト入力コンポーネントに適用します。

## 前提条件

- フォームが有効になっている Adobe Experience Manager インスタンスへのアクセス。
- テンプレートを編集する権限（通常は `template-authors` グループの一部）。
- アダプティブフォームを作成および編集する権限。

## パート 1：テンプレートで検証パターンを有効にする

>[!VIDEO](https://video.tv.adobe.com/v/3480390/)

### 手順 1：テンプレートコンソールにアクセスする

1. Adobe Experience Manager のホーム画面で、左側のサイドバーにある&#x200B;**ツール**（ハンマーアイコン）をクリックします。
2. **一般／テンプレート**&#x200B;に移動します。
3. プロジェクトテンプレートを含むフォルダー（例：コアコンポーネントの例）を選択します。
4. 変更する特定のテンプレート（例：AF Blank）を見つけて選択します。
5. **編集**&#x200B;をクリックします（またはテンプレートを直接開きます）。

### 手順2：コンポーネント構造へアクセスする

1. **構造**&#x200B;ビューにいることを確認します（必要に応じて、右上のモードドロップダウンから「構造」を選択します）。
2. メインの&#x200B;**レイアウトコンテナ**&#x200B;またはテキスト入力コンポーネントが許可されている特定のコンテナを探します。
3. コンテナをクリックして、許可されたコンポーネントリストを表示します。
4. リストから&#x200B;**アダプティブフォームのテキストボックス**&#x200B;コンポーネントを見つけます。
5. コンポーネント名の横にある&#x200B;**ポリシー**&#x200B;アイコン（ファイル/引き出し記号で表されます）をクリックします。

### 手順3：テキスト入力コンポーネントポリシーを設定する

1. ポリシーエディターで、ウィンドウ右側にある「**フォーマット**」タブをクリックします。
2. 使用可能な検証パターンのリストが表示されます。有効にする形式のチェックボックスをオンにします。
   - 電話番号
   - 社会保障番号
   - メール英数字
   - 郵便番号

### 手順4：ポリシーを定義し保存する

1. 「**ポリシー**」タブに切り替えます（または左側の設定を確認してください）。
2. 「**ポリシータイトル**」フィールドに、わかりやすい名前（例：`new-textinput-default-policy`）を入力します。
3. 変更を保存するには、右上隅にある「**完了**」（チェックマーク）アイコンをクリックします。

## パート 2：フォームを作成し検証パターンを使用する

テンプレートで検証パターンが有効になったので、アダプティブフォームを作成してテキスト入力コンポーネントに適用できます。

>[!VIDEO](https://video.tv.adobe.com/v/3480391/)

### 手順 3：アダプティブフォームを作成する

1. **フォームとドキュメント**&#x200B;インターフェイスに移動します。
2. 右上隅の「**作成**」ボタンをクリックします。
3. ドロップダウンオプションから「**アダプティブフォーム**」を選択します。
4. 検証パターンを有効にしたテンプレート（例：**空白**）を選択し、「**次へ**」をクリックします。
5. 「プロパティを追加」画面で、次の操作を行います。
   - フォームの&#x200B;**タイトル**&#x200B;に名前（例：&quot;testPolicy&quot;）を入力します。「**名前**」フィールドは、タイトルに基づいて自動的に入力されます。
   - 正しい&#x200B;**テーマクライアントライブラリ**&#x200B;が選択されていることを確認します（例：`adaptiveform.theme.canvas3`）。
6. 「**作成**」をクリックします。 成功メッセージが表示されます。
7. 「**編集**」をクリックして、フォームを編集モードで開きます。

### 手順2：テキスト入力コンポーネントを追加する

1. フォームエディターで、左側のサイドバーにある&#x200B;**コンポーネント**&#x200B;ブラウザーを見つけます。
2. 検索バーを使用して、**アダプティブフォームのテキストボックス**&#x200B;コンポーネントを検索します。&quot;text&quot; と入力すると、リストがフィルタリングされます。
3. 「**アダプティブフォームのテキストボックス**」コンポーネントをクリックして、メインキャンバス領域にドラッグします。

### 手順3：コンポーネントの書式を設定する

1. 新しく追加した&#x200B;**テキスト入力**&#x200B;コンポーネントをクリックして選択します。
2. コンポーネントの上に表示されるツールバーの「**設定**」アイコン（レンチで表されます）をクリックします。
3. プロパティダイアログで、「**形式**」タブに移動します。
4. **タイプ**&#x200B;ドロップダウンメニューを探し、選択を&#x200B;**電話番号**（または別の有効な形式）に変更します。

### 手順4：データ検証を設定する

1. コンポーネント設定ダイアログ内で、「**検証**」タブに移動します。
2. 「**検証パターン**」セクションを探します。
3. 「**タイプ**」ドロップダウンメニューをクリックします。
4. 検証パターンのリストから&#x200B;**電話番号**&#x200B;を選択します。これにより、ユーザーが有効な電話番号の形式と一致しないテキストを入力すると、フォームにエラーが発生します。
5. 変更を保存するには、ダイアログボックス右上にある「**完了**」（チェックマーク）アイコンをクリックします。

## 検証

検証パターンが正しく機能していることを確認するには、次の操作を行います。

1. エディターでフォームをプレビューします。
2. **テキスト入力**&#x200B;コンポーネントにテストデータを入力します。
3. 次のことを確認します。
   - 有効な検証パターンは、コンポーネントの「フォーマット」タブと「検証」タブに表示されます。
   - 無効な入力が行われると、適切なエラーメッセージが表示されます。
   - 有効な入力はエラーなしで受け付けられます。

## トラブルシューティング

- **問題：**&#x200B;フォームコンポーネントに検証パターンが表示されません。
   - **解決策：**&#x200B;テンプレート ポリシーが正しく保存されたこと、およびフォームの作成時に正しいテンプレートを使用していることを確認してください。

- **問題：**&#x200B;テンプレートエディターにアクセスできません。
   - **解決策：**&#x200B;テンプレートを編集するために必要な権限を持っていることを確認します（`template-authors` グループのメンバーシップ）。

- **問題：**&#x200B;テキスト入力コンポーネントが入力を正しく検証しません。
   - **解決策：**&#x200B;コンポーネントの「検証」タブで検証パターンの設定を再確認し、正しいパターンが選択されていることを確認します。

## 次の手順

Adobe Experience Manager アダプティブフォームでテキスト入力検証パターンを有効にして使用した後は、次の操作を行います。

- 他のデータタイプに対して追加の検証パターンを設定します。
- その他のコンポーネントポリシーを確認して、フォームの動作をカスタマイズします。
- フォーム送信処理とデータ処理ロジックを設定します。
